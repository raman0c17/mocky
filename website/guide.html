Installation

Mocky needs Python 3.9+ and runs on Windows, macOS and Linux.

From PyPI

pip install mocky

From source (for development)

git clone https://github.com/raman0c17/mocky.git
cd mocky
python3 -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -e ".[dev]"

Seeing externally-managed-environment? That's a macOS/Homebrew (PEP 668) guard. Install inside the virtual environment shown above rather than using --break-system-packages.

Quickstart

Generate your first deck in two commands:

pip install mocky
mocky

Choose “Generate a presentation from an idea”, type your idea, pick a template, and Mocky writes a .pptx into presentations/.

A typical mocky session

Using the CLI

Running mocky opens an interactive menu with two paths:

  • Generate from an idea — scaffolds Markdown into generated_markdowns/, writes agent prompts to prompt_outputs/, and renders a .pptx into presentations/.
  • Convert existing Markdown — drop a .md file into input_files/ and convert it directly.

Using the library

Workflow A — from an idea

from mocky import TemplateLibrary, MarkdownParser, PowerPointGenerator

idea = "Launching Mocky 1.0 to the open-source community"
markdown = TemplateLibrary.build_markdown(idea, "product-launch")
path = TemplateLibrary.save_markdown(markdown, "generated_markdowns", "launch.md")

parsed = MarkdownParser().parse_markdown(path)
PowerPointGenerator().create_presentation(parsed["slides"], "presentations/launch.pptx")

Workflow B — from existing Markdown

from mocky import MarkdownParser, PowerPointGenerator

parsed = MarkdownParser().parse_markdown("input_files/my_outline.md")
PowerPointGenerator().create_presentation(parsed["slides"], "presentations/out.pptx")

The slide model

Under the hood a deck is just a list of slide dicts:

slides = [
    {"title": "Slide title", "content": ["A bullet", "Another bullet"]},
    {"title": "With a picture", "content": [{"image": "/path/to/local.png"}]},
    {"title": "With a link", "content": [{"link": "https://example.com"}]},
]
PowerPointGenerator().create_presentation(slides, "presentations/manual.pptx")

Markdown maps to this model as: the first # is the deck title, every ## starts a new slide, and paragraphs, lists, images and links become content items.

AI-agent prompts

Mocky generates ready-to-paste prompts for 15 agents/CLIs — it never calls them itself, so your credentials stay local.

from mocky import AgentPromptBuilder

AgentPromptBuilder.list_agents()      # 15 agents
AgentPromptBuilder.build_prompt("claude code", "My idea", "product-launch")

Keep secrets out of git. Copy .env.example to .env and fill in only the keys you use — .env is git-ignored by default.

Docker

docker build -t mocky .

docker run --rm -it \
  -v "$PWD/input_files:/app/input_files" \
  -v "$PWD/presentations:/app/presentations" \
  mocky

Because the stack is pure Python, the same image builds for linux/amd64 and linux/arm64 with docker buildx.

Contributing

Contributions are welcome! Set up a dev environment, then run the checks before opening a PR:

pip install -e ".[dev]"
ruff check .
ruff format --check .
pytest
  • Keep changes cross-platform — no OS-specific deps on the default path.
  • Follow Conventional Commits.
  • Good first issues: add a template, add an agent, or improve these docs.

FAQ

Do I need Microsoft PowerPoint or Office?

No. Mocky renders with python-pptx, which is pure Python.

Which operating systems are supported?

Windows (x64/ARM), macOS (Intel/Apple Silicon) and Linux — the same code path everywhere.

Does Mocky call AI services for me?

No. It only generates prompt text for you to paste into the agent of your choice.

Where do generated files go?

generated_markdowns/, prompt_outputs/ and presentations/ — all git-ignored by default.
